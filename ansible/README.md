# Ansible Configuration

This documentation covers the Ansible configuration located in the `ansible/` directory to configure the application servers after they are provisioned by Terraform.

## Overview

Two playbooks are provided:

- **`site.yml`** (uses the `docker_engine` role): Installs Docker Engine, enables the service, and adds `ec2-user` to the `docker` group.
- **`site-nginx.yml`** (uses the `webserver` role): Installs **Git** and **Nginx**, deploys the static site, and configures Nginx from a custom template.

Both playbooks target the `[ec2_instances]` host group.

## Directory Structure

- `inventory-<env>.ini`: Defines the target hosts and connection details. **Generated by Terraform** during `apply` from `terraform/environments/<env>/ansible_inventory.tpl`; do not edit by hand.
- `site.yml`: Playbook that installs Docker.
- `site-nginx.yml`: Playbook that installs and configures Nginx.
- `roles/`: Contains reusable Ansible roles.
  - `docker_engine/`: Installs and enables Docker Engine.
  - `webserver/`: Handles Nginx installation, configuration, and git cloning.

## Prerequisites

- Ansible installed on your local machine.
- SSH access to the EC2 instance (ensure your public key is added to the instance via Terraform or manually).
- The private key file located at `~/.ssh/id_ed25519` (or update `inventory-dev.ini` to match your key path).

## Usage

1.  **Inventory**:
    The environment-specific inventory file (`inventory-<env>.ini`) is generated by Terraform during `apply`. It looks like:

    ```ini
    [ec2_instances]
    <EC2_PUBLIC_IP> ansible_user=ec2-user ansible_ssh_private_key_file=~/.ssh/id_ed25519
    ```

    If running outside the Terraform workflow, create this file manually using the public IPs from the Terraform `ec2_public_ip` output.

2.  **Run a Playbook**:
    Navigate to the `ansible/` directory and execute:

    ```bash
    ansible-playbook -i inventory-dev.ini site.yml          # install Docker
    ansible-playbook -i inventory-dev.ini site-nginx.yml    # install/configure Nginx
    ```

3.  **Verify**:
    Open a web browser and navigate to the EC2 instance's public IP (http://<EC2_PUBLIC_IP>) to see the deployed portfolio.
    *Note: Ensure the Security Group allows inbound traffic on port 80.*
